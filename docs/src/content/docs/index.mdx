shux is an experimental terminal multiplexer with a tmux-like attach flow, Neovim-style Lua configuration, and a resurrection layer that records pane output so you can reattach closer to the state you left.

:::caution[Experimental]
shux is under active development. Behavior and configuration may change between releases.
:::

## What makes shux different

<CardGrid stagger>
	<Card title="Resurrection" icon="rocket">
		Detach and come back later. shux journals PTY output and layout so reattach can restore scrollback and pane structure.
	</Card>
	<Card title="Lua configuration" icon="setting">
		Configure options, keymaps, and plugins with a Neovim-inspired `~/.config/shux/` layout.
	</Card>
	<Card title="Daemon + clients" icon="laptop">
		One local daemon holds your sessions. Clients attach over SSH to the same backend you can detach from and rejoin.
	</Card>
	<Card title="tmux-like keys" icon="open-book">
		Familiar prefix bindings for split, window, and pane navigation — customizable in Lua.
	</Card>
</CardGrid>
